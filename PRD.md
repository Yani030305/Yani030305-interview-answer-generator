# HireMind AI 面试助手 产品需求文档

## 1. 产品概览

HireMind AI 面试助手是一款基于 AI 技术的面试准备工具，旨在帮助用户快速生成个性化的面试回答，提高面试成功率。

- **核心价值**：通过分析用户上传的简历和文档，结合 AI 技术，生成符合用户背景和岗位需求的个性化面试回答。
- **目标用户**：即将参加面试的求职者，包括校招生和社招人员。
- **解决问题**：减轻用户准备面试的时间和压力，提供专业、个性化的面试回答参考。

## 2. 核心功能

### 2.1 功能模块

HireMind AI 面试助手包含以下核心功能模块：

1. **用户管理**：注册、登录、个人信息管理、单设备登录限制、邮箱和手机号唯一约束
2. **文档管理**：上传、查看、删除文档
3. **问题管理**：浏览、搜索、筛选面试问题
4. **回答生成**：AI 生成个性化回答
5. **回答管理**：编辑、复制、重新生成回答
6. **历史记录**：查看、管理历史生成记录
7. **导出功能**：导出回答为文档
8. **积分系统**：积分管理、充值、消费、积分使用记录
9. **支付系统**：ZPAY 支付集成、订单管理、充值历史
10. **批量生成**：一键生成全部回答，享受优惠价格
11. **问题反馈**：提交功能建议、Bug 报告或其他问题
12. **CUHK邮箱赠送**：香港中文大学邮箱注册赠送500积分

## 3. 核心流程

### 用户操作流程

1. **注册登录**：用户注册并登录系统
2. **上传文档**：上传简历、作品集等文档
3. **选择模式**：选择校招生或社招模式
4. **浏览问题**：浏览问题列表，选择感兴趣的问题
5. **生成回答**：点击生成按钮，AI 生成个性化回答（消耗积分）
6. **编辑优化**：编辑和优化生成的回答
7. **管理历史**：查看和管理历史生成记录
8. **导出文档**：导出所有问答为文档（消耗积分）
9. **批量生成**：使用一键生成全部回答功能，一次性生成所有问题的回答（享受优惠价格）
10. **问题反馈**：提交功能建议、Bug 报告或其他问题
11. **充值积分**：当积分不足时，进行积分充值
12. **支付订单**：选择套餐，生成支付订单，扫码支付

### 系统流程

1. **用户认证**：验证用户身份，获取用户信息，检查单设备登录
2. **文档处理**：解析上传的文档，提取文本内容
3. **问题匹配**：根据用户模式和文档内容，推荐相关问题
4. **AI 生成**：调用 DeepSeek API 生成个性化回答
5. **批量生成**：处理批量生成请求，按顺序生成所有问题的回答
6. **历史记录**：保存生成的回答到数据库，支持去重
7. **积分管理**：扣除生成回答和导出文档的积分，支持失败回滚
8. **订单处理**：创建订单，调用 ZPAY 支付接口，处理支付回调
9. **导出处理**：将回答转换为 DOCX 或 Markdown 格式
10. **反馈处理**：保存用户提交的问题反馈到数据库

## 4. 用户界面设计

- **主色调**：蓝色系（代表专业、信任）
- **辅助色**：绿色（成功）、黄色（警告）、红色（错误）
- **按钮风格**：圆角矩形，有轻微的阴影效果

## 5. 技术方案

### 5.1 技术栈

- **前端**：Next.js 14, React, TypeScript, Tailwind CSS
- **后端**：Supabase (数据库、认证、边缘函数)
- **AI 服务**：DeepSeek API
- **支付服务**：ZPAY
- **工具库**：lucide-react, @radix-ui, docx.js

### 5.2 数据结构

1. **用户表** (profiles)
   - id: string (用户ID)
   - email: string (邮箱)
   - phone_number: string (手机号)
   - full_name: string (姓名)
   - credits: number (积分)
   - session_id: string (会话ID，用于单设备登录)
   - created_at: timestamp (创建时间)
   - updated_at: timestamp (更新时间)

2. **文档表** (documents)
   - id: string (文档ID)
   - user_id: string (用户ID)
   - name: string (文件名)
   - type: string (文件类型)
   - size: number (文件大小)
   - extracted_text: text (提取的文本)
   - uploaded_at: timestamp (上传时间)

3. **回答历史表** (answer_history)
   - id: string (历史记录ID)
   - user_id: string (用户ID)
   - question_id: string (问题ID)
   - answer_zh: text (中文回答)
   - answer_en: text (英文回答)
   - created_at: timestamp (创建时间)
   - updated_at: timestamp (更新时间)

4. **积分交易表** (credit_transactions)
   - id: string (交易ID)
   - user_id: string (用户ID)
   - amount: number (金额)
   - type: string (类型：购买、消费、退款)
   - description: string (描述)
   - created_at: timestamp (创建时间)

5. **积分套餐表** (credit_packages)
   - id: string (套餐ID)
   - package_id: string (套餐标识)
   - name: string (套餐名称)
   - credits: number (积分数量)
   - price: number (价格)
   - original_price: number (原价)
   - bonus: number (赠送积分)
   - is_popular: boolean (是否热门)
   - is_active: boolean (是否激活)
   - sort_order: number (排序)

6. **订单表** (orders)
   - id: string (订单ID)
   - user_id: string (用户ID)
   - package_id: string (套餐ID)
   - package_name: string (套餐名称)
   - price: number (价格)
   - credits: number (积分数量)
   - status: string (状态：pending、paid、failed、refunded)
   - payment_provider: string (支付提供商)
   - payment_order_id: string (支付订单ID)
   - error_message: string (错误信息)
   - created_at: timestamp (创建时间)
   - updated_at: timestamp (更新时间)
   - paid_at: timestamp (支付时间)

7. **请求日志表** (request_logs)
   - id: string (日志ID)
   - user_id: string (用户ID)
   - endpoint: string (接口路径)
   - method: string (请求方法)
   - ip_address: string (IP地址)
   - user_agent: string (用户代理)
   - request_body: json (请求体)
   - response_status: number (响应状态码)
   - response_time_ms: number (响应时间)
   - error_message: string (错误信息)
   - created_at: timestamp (创建时间)

8. **反馈表** (feedback)
   - id: string (反馈ID)
   - user_id: string (用户ID，可为空)
   - user_email: string (用户邮箱，可为空)
   - type: string (反馈类型：suggestion、bug、other)
   - content: text (反馈内容)
   - created_at: timestamp (创建时间)

### 5.3 API 接口

1. **认证接口**
   - POST /api/auth/register - 用户注册
   - POST /api/auth/login - 用户登录
   - POST /api/auth/logout - 用户登出

2. **文档接口**
   - POST /api/parse-document - 上传和解析文档
   - GET /api/documents - 获取用户文档列表
   - DELETE /api/documents/:id - 删除文档

3. **回答接口**
   - POST /api/generate-answer - 生成回答
   - GET /api/answer-history - 获取历史记录
   - DELETE /api/answer-history/:id - 删除历史记录

4. **积分接口**
   - GET /api/credits - 获取用户积分
   - GET /api/credit-packages - 获取积分套餐列表
   - GET /api/credit-history - 获取积分使用记录

5. **订单接口**
   - POST /api/orders - 创建订单
   - GET /api/orders - 获取订单列表
   - GET /api/orders?orderId=xxx - 获取订单详情

6. **支付接口**
   - POST /api/payment/callback - 支付回调处理

7. **导出接口**
   - POST /api/export - 导出文档

8. **反馈接口**
   - POST /api/feedback - 提交问题反馈

## 6. 数据需求

### 6.1 数据存储

- **用户数据**：存储用户基本信息、积分余额、会话ID、手机号
- **文档数据**：存储用户上传的文档信息和提取的文本
- **回答数据**：存储生成的回答历史
- **交易数据**：存储积分交易记录，包括充值、消费、赠送等
- **订单数据**：存储订单信息和支付状态
- **日志数据**：存储请求日志，用于分析和排查问题
- **反馈数据**：存储用户提交的问题反馈

### 6.2 数据安全

- **加密存储**：用户敏感信息加密存储
- **访问控制**：基于用户ID的数据访问控制
- **数据备份**：定期备份数据库
- **合规性**：符合数据隐私法规
- **单设备登录**：通过 session_id 实现单设备登录限制

## 8. 风险评估

### 8.1 技术风险

- **AI 服务依赖**：DeepSeek API 服务不稳定或价格变动
- **文档解析**：复杂文档解析失败或提取效果不佳
- **性能问题**：大量文档处理导致系统响应缓慢
- **支付风险**：支付接口调用失败或回调处理异常
- **批量生成**：批量生成过程中出现异常导致部分回答生成失败

### 8.2 业务风险

- **用户体验**：AI 生成的回答质量不符合用户期望
- **积分系统**：积分定价不合理导致用户流失
- **竞争风险**：市场上出现类似产品
- **支付风险**：支付安全问题或订单异常
- **批量生成**：批量生成价格过高导致用户使用率低

### 8.3 解决方案

- **技术风险**：
  - 实现 AI 服务的降级方案
  - 优化文档解析算法
  - 实现文档处理队列和缓存机制
  - 实现支付重试机制和订单状态同步
  - 实现批量生成的错误处理和重试机制

- **业务风险**：
  - 持续优化 AI 模型和提示词
  - 进行市场调研，合理定价
  - 持续迭代，增加差异化功能
  - 完善支付流程和用户提示
  - 为批量生成设置优惠价格，提高用户使用率
